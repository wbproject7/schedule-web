## 1. 접속 방법

웹 브라우저(Chrome, Safari, Edge 등)를 열고 전달받은 주소로 접속합니다.

| 환경 | 접속 주소 예시 |
|------|---------------|
| 같은 컴퓨터에서 실행 중 | `http://localhost:5000` |
| ngrok으로 외부 공유 | `https://xxxxx.ngrok-free.app` |
| Render.com에 배포됨 | `https://schedule-generator-xxxx.onrender.com` |

> ngrok 주소로 접속 시 첫 화면에 경고 페이지가 나타날 수 있습니다. **"Visit Site"** 버튼을 눌러 진행하세요.

## 2. 비밀번호 입력

접속하면 가장 먼저 비밀번호 입력 화면이 나타납니다.

- 관리자로부터 전달받은 비밀번호를 입력합니다.
- **확인** 버튼을 클릭하거나 Enter 키를 누릅니다.
- 인증이 완료되면 "인증 완료" 메시지가 표시되며 스케줄 생성을 시작할 수 있습니다.

> 비밀번호 인증은 24시간 동안 유지됩니다. 이후에는 다시 입력해야 합니다.

## 3. Step 1: 기간 선택

스케줄을 만들 **연도**와 **월**을 선택합니다.

- **연도**: 드롭다운에서 선택 (2025~2028년)
- **월**: 드롭다운에서 선택 (1~12월)
- **확인** 버튼 클릭

예시: 2026년 4월 스케줄을 만들려면 `2026년` `4월` 선택 후 확인

> 현재 날짜의 연/월이 기본값으로 설정되어 있습니다.

---

## 4. Step 2: 직원 명단 입력

스케줄에 포함될 직원 이름을 입력합니다.

- 직원 이름을 **쉼표(,)로 구분**하여 한 줄에 입력합니다.
- 최소 **2명**, 최대 **50명**까지 가능합니다.
- 이름은 **20자 이내**로 입력해주세요.
- **같은 이름이 중복되면 안 됩니다.** (동명이인은 구분 가능하게 입력)

**입력 예시:**
```
김철수, 이영희, 박민수, 정하나, 최동욱
```

> 이름 순서가 사전 신청(D/O, M/O) 충돌 시 우선순위에 영향을 줍니다. (먼저 입력된 직원이 우선)

---

## 5. Step 3: 관리자 지정

관리자(매니저)를 지정합니다. **관리자끼리는 같은 날 동시에 쉬지 않습니다.**

- 직원 이름이 버튼으로 표시됩니다.
- 관리자에 해당하는 직원을 **클릭**하면 선택됩니다 (파란색 강조).
- 다시 클릭하면 선택이 해제됩니다.
- 관리자가 없으면 아무것도 선택하지 않고 **확인** 클릭

**관리자 지정 효과:**
- 하루에 관리자 중 최대 1명만 휴무 가능
- 항상 최소 1명의 관리자가 근무하도록 보장

---

## 6. Step 4: 공휴일 선택

해당 월의 공휴일을 달력에서 선택합니다.

- 달력의 날짜를 **클릭**하면 공휴일로 지정됩니다 (노란색 표시).
- 다시 클릭하면 **해제**됩니다.
- 주말(토/일)은 이미 회색으로 표시되어 있으며, 별도로 선택할 필요 없습니다.
- 공휴일이 없으면 아무것도 선택하지 않고 **확인** 클릭

**공휴일 지정 효과:**
- 해당 날은 주말과 동일하게 취급됩니다.
- "주말/공휴일 최소 근무 인원" 제약이 적용됩니다.

> 대체 공휴일, 임시 공휴일 등도 이 단계에서 선택하세요.

---

## 7. Step 5: 제약 조건 설정

스케줄 생성의 규칙을 설정합니다. 기본값이 입력되어 있으며, 필요시 수정합니다.

### 각 항목 설명

| 항목 | 기본값 | 설명 |
|------|--------|------|
| **월 휴무(D/O) 횟수** | 8회/인 | 한 달 동안 각 직원에게 배정할 지정 휴무(D/O) 횟수. 모든 직원에게 동일하게 적용됩니다. |
| **최대 연속 근무** | 5일 | 연속으로 일할 수 있는 최대 일수. 예: 5일 설정 시 6일 연속 근무는 불가합니다. |
| **최대 연속 휴무** | 4일 | 연속으로 쉴 수 있는 최대 일수 (D/O + M/O 합산). 예: 4일 설정 시 5일 연속 휴무는 불가합니다. |
| **평일 최소 근무** | (인원-3)명 | 평일에 반드시 출근해야 하는 최소 인원 수. |
| **주말/공휴일 최소 근무** | (인원-1)명 | 주말 및 공휴일에 반드시 출근해야 하는 최소 인원 수. |
| **평일 최소 휴무** | 2명 | 평일에 반드시 쉬어야 하는 최소 인원 수 (D/O + M/O 합산). |

### 설정 팁

- **D/O 횟수**가 너무 크면 근무 인원이 부족해질 수 있습니다.
- **최소 근무 인원**이 전체 인원과 같거나 크면 솔버가 실패합니다.
- 일반적으로 기본값 그대로 사용해도 무방합니다.

## 8. Step 6: 사전 신청 (D/O / M/O)

각 직원별로 미리 확정된 휴무(D/O)나 연차(M/O)를 입력합니다.

### 화면 구성

- 상단에 직원 이름 탭이 표시됩니다 (현재 직원은 파란색, 완료된 직원은 초록색).
- 달력이 표시되며 날짜를 클릭하여 신청합니다.

### 달력 클릭 방법

| 클릭 횟수 | 동작 | 색상 |
|-----------|------|------|
| **1번 클릭** | D/O (지정 휴무) 선택 | 파란색 |
| **2번 클릭** | M/O (연차) 변경 | 빨간색 |
| **3번 클릭** | 선택 해제 | 원래 색상 |

### 진행 방법

1. 첫 번째 직원의 신청 사항을 달력에서 선택
2. **다음 직원 >** 버튼 클릭
3. 모든 직원을 순서대로 진행
4. 마지막 직원까지 완료하면 자동으로 다음 단계로 이동

> 사전 신청이 없는 직원은 아무것도 선택하지 않고 **다음 직원 >**을 누르면 됩니다.

### 주의사항

- 같은 날에 너무 많은 직원이 휴무를 신청하면 **충돌**이 발생합니다.
- 충돌 시 입력 순서(Step 2에서 입력한 순서)가 빠른 직원이 우선됩니다.
- 충돌이 발생해도 솔버가 자동으로 조정하므로, 결과 화면에서 확인하면 됩니다.

---

## 9. Step 7: 주말 형평성

주말/공휴일의 D/O(휴무)를 공평하게 분배할지 선택합니다.

| 옵션 | 설명 |
|------|------|
| **적용** (기본) | 모든 직원에게 주말/공휴일 D/O를 최소 1회 배정하고, 한 직원에게 과도하게 몰리지 않도록 합니다. |
| **미적용** | 주말/공휴일 D/O를 솔버가 자유롭게 배정합니다. (특정 직원에게 몰릴 수 있음) |

> 대부분의 경우 **적용**을 권장합니다.

---

## 10. Step 8: 최종 확인 및 생성

지금까지 입력한 모든 내용이 요약되어 표시됩니다.

### 확인 항목

- 대상 기간 (연/월, 총 일수)
- 직원 수
- 관리자 목록
- 주말/공휴일 수
- 제약 조건 (D/O 횟수, 연속 근무, 근무 인원 등)
- 각 직원별 사전 신청 내역

### 버튼

| 버튼 | 동작 |
|------|------|
| **스케줄 생성하기** | 솔버를 실행하여 스케줄을 생성합니다 (최대 30초 소요). |
| **처음부터** | 모든 입력을 초기화하고 Step 1부터 다시 시작합니다. |

> 내용을 꼼꼼히 확인한 후 생성 버튼을 눌러주세요.

## 11. 결과 확인

스케줄 생성이 완료되면 아래 내용이 순서대로 표시됩니다.

### 충돌 감지 알림 (해당 시)

사전 신청이 충돌한 경우, 어떤 날에 누구의 신청이 조정되었는지 표시됩니다.
선착순(입력 순서)으로 자동 조정되므로 별도 조치는 필요 없습니다.

### 스케줄 표

가로로 스크롤 가능한 표가 표시됩니다.

- **행(가로)**: 각 직원
- **열(세로)**: 날짜 (1일~말일)
- **셀 내용**: W, D/O, M/O 중 하나
- **마지막 행**: `[근무인원]` - 해당 날의 총 근무 인원
- **우측 통계**: 각 직원의 W(근무), D/O(휴무), M/O(연차) 합계

**색상 구분:**

| 색상 | 의미 |
|------|------|
| 흰색 | W (근무) |
| 파란색 배경 | D/O (지정 휴무) |
| 빨간색 배경 | M/O (연차) |

### 제약 조건 검증 결과

설정한 모든 제약 조건이 올바르게 적용되었는지 검증 결과를 보여줍니다.

- 체크 표시: 해당 조건 충족
- X 표시: 해당 조건 미충족 (일반적으로 발생하지 않음)

검증 항목 예시:
- D/O 8회/인
- 최소 근무 인원
- 연속 근무 5일 이하
- 주말/공휴일 D/O 분배
- 평일 최소 2명 휴무
- 연속 휴무 4일 이하
- 관리자 휴무 비겹침
- 평일 D/O 분산

## 12. Excel/CSV 다운로드

결과 화면 하단에 두 가지 다운로드 버튼이 있습니다.

| 버튼 | 설명 |
|------|------|
| **Excel 다운로드** | `.xlsx` 파일. 색상 서식이 적용되어 있어 보기 편합니다. |
| **CSV 다운로드** | `.csv` 파일. 다른 프로그램에서 열 수 있는 범용 형식입니다. |

### Excel 파일 특징

- 셀 색상: D/O는 파란색, M/O는 빨간색으로 구분
- 헤더: 파란색 배경에 흰색 글씨
- 셀 테두리 및 정렬 적용
- 시트 이름: `2026년4월` 형식

### 다운로드 주의사항

- 파일은 서버에서 **1시간 동안** 보관됩니다.
- 1시간이 지나면 다시 스케줄을 생성해야 합니다.
- 필요한 파일은 즉시 다운로드하세요.

## 13. 용어 설명

| 용어 | 영문 | 의미 |
|------|------|------|
| **W** | Work | 근무일. 해당 날에 출근합니다. |
| **D/O** | Day Off | 지정 휴무. 솔버가 배정하거나 사전에 신청한 휴무입니다. 월 D/O 횟수에 포함됩니다. |
| **M/O** | Mandatory Off (연차) | 연차 휴가. 사전 신청으로만 지정되며, 반드시 해당 날 휴무합니다. D/O 횟수에 포함되지 않습니다. |
| **솔버** | Solver | OR-Tools라는 최적화 엔진. 모든 제약 조건을 만족하는 최적의 스케줄을 자동으로 찾아줍니다. |
| **형평성** | Fairness | 주말/공휴일 D/O를 직원들에게 공평하게 배분하는 기능입니다. |
| **연속 근무** | Consecutive Work | 쉬지 않고 연속으로 근무하는 일수입니다. |
| **연속 휴무** | Consecutive Off | 연속으로 쉬는 일수입니다 (D/O + M/O 합산). |

## 14. FAQ / 주의사항

### Q1: "스케줄을 찾을 수 없습니다"라고 나옵니다.

솔버가 모든 제약 조건을 동시에 만족하는 스케줄을 찾지 못한 경우입니다.

**해결 방법:**
- 최소 근무 인원을 줄여보세요 (예: 평일 4명 -> 3명)
- D/O 횟수를 줄여보세요 (예: 8회 -> 7회)
- M/O(연차) 신청을 줄여보세요 (특히 같은 날에 집중된 경우)
- 관리자 수를 줄여보세요 (관리자가 3명 이상이면 제약이 강해집니다)
- 최대 연속 근무/휴무 제한을 완화해보세요

### Q2: 브라우저를 새로고침하면 데이터가 사라지나요?

아닙니다. 각 단계를 진행할 때마다 **자동 저장**됩니다.

- 새로고침 시 "이전 작업이 저장되어 있습니다" 메시지가 표시됩니다.
- **이어서 진행** 버튼을 누르면 마지막 진행 단계부터 재개됩니다.
- **새로 시작** 버튼을 누르면 처음부터 다시 시작합니다.
- 저장 데이터는 **24시간** 후 자동으로 만료됩니다.

### Q3: 같은 날에 여러 직원이 D/O를 신청하면 어떻게 되나요?

최소 근무 인원을 유지할 수 있는 범위 내에서 신청이 수락됩니다.

- 수용 가능 인원을 초과하면 **선착순**(직원 입력 순서)으로 자동 조정됩니다.
- 결과 화면 상단에 충돌 및 조정 내역이 표시됩니다.

### Q4: D/O와 M/O의 차이가 뭔가요?

| 구분 | D/O (지정 휴무) | M/O (연차) |
|------|----------------|-----------|
| 배정 방식 | 솔버가 자동 배정 + 사전 신청 가능 | 사전 신청만 가능 |
| 월 횟수 제한 | 있음 (예: 8회) | 없음 (신청한 만큼) |
| 변경 가능 여부 | 솔버가 조정 가능 | 반드시 해당 날 휴무 (변경 불가) |

쉽게 말해:
- **D/O**: "이 날 쉬고 싶어요" (조정될 수 있음)
- **M/O**: "이 날 반드시 쉽니다" (확정)

### Q5: 스케줄 생성에 시간이 얼마나 걸리나요?

- 보통 **5~15초** 이내에 완료됩니다.
- 직원 수가 많거나(15명+) 제약이 까다로우면 최대 **30초**까지 소요될 수 있습니다.
- 30초 초과 시 타임아웃으로 실패하며, 제약 조건을 완화하면 해결됩니다.

### Q6: 결과가 마음에 들지 않으면 다시 생성할 수 있나요?

네. 결과 화면 하단의 **새 스케줄 만들기** 버튼을 누르면 처음부터 다시 시작할 수 있습니다. 같은 조건으로 다시 생성하면 솔버의 특성상 약간 다른 결과가 나올 수 있습니다.

### Q7: 모바일에서도 사용할 수 있나요?

네. 스마트폰, 태블릿의 모바일 브라우저에서도 사용 가능합니다. 다만 스케줄 결과 표는 가로 스크롤이 필요하므로 PC/태블릿 사용을 권장합니다.

### Q8: 인증이 만료되었다는 메시지가 나옵니다.

비밀번호 인증은 **24시간** 유효합니다. 24시간이 지나면 다시 비밀번호를 입력해야 합니다. 스케줄 생성 도중 만료되면 재로그인 후 저장된 데이터로 이어서 진행할 수 있습니다.

### Q9: 분당 요청 제한이 있나요?

네. 서버 부하 방지를 위해 **1분에 최대 10회** 요청으로 제한되어 있습니다. "요청이 너무 많습니다" 메시지가 나타나면 1분 후 다시 시도하세요.

### Q10: 다운로드한 Excel 파일이 깨져 보입니다.

- 파일을 Microsoft Excel, Google Sheets, Numbers 등으로 열어보세요.
- CSV 파일의 경우 인코딩이 UTF-8(BOM)이므로 대부분의 프로그램에서 한글이 정상 표시됩니다.
- 그래도 문제가 있으면 CSV 파일을 사용해주세요.
